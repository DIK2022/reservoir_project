# OLAP-Reservoir-Decision-Support-System

#1. Цели и задачи

Разработать десктопное приложение для финансового или логистического анализа с использованием:

    - OLAP-куба (анализ многомерных данных).
    - Резервуарного компьютера (Reservoir Computing) с использованием Эхосетевого подхода, ESN.
    - Эвалюционной архитектуры.
    - GUI на PySide6 (дизайн в Qt Disigner).
    - Высокопроизводительных модулей на Rust для численного ядра.
    - Поддержка прямого запуска .ui и скомпилированного .py.
    - Работа на ноутбуке (1.8 GHz, 16GB RAM) — оптимизация под CPU
    
#2. Функциональные тредования
#2.1. OLAP
    - Источник данных: CSV / Parquet (финанчы: сделки, котировки; логистика: поставки, маршруты).
    - Измерения: время, контрагент, продукт, регион.
    - Меры: сумма, количество, рейтинг.
    - Поддержка slice/dice, drilldown/up.

#2.2. Резервуарный компьютер
    - Реализация эхосети (Echo State Network) с хаотическим резервуаром (спектральный радиус <1, случайные веса).
    - Обучение: гребневая регрессия (Ridge) на выходных весах.
    - Эвалюционная архитектура: смена резервуара при ухудшении ошибки (мутация спектрального радиуса, плотности связей).
    - Прогнозирование временных рядов (цена акций, загрузка складов).

#2.3. Взаимодействие

    - Пользователь загружает данные -> строит OLAP-срез -> отправляет временной ряд в резервуар -> получает прогноз.
    - Можно менять гиперпараметры резервуара (size, leaking rate, ridge alpha).

#2.4. Интерфейс
    - Главное окно: таблица OLAP + график matplotlib
    - Панель управления резервуаром.
    Вывод ошибки прогноза (NMSE).

#3. Архитектура
    - Rust (crate reservoir_core) - вычисления резервуара, хаотическая динамика, обучение.
    - Python - PySide6, OLAP (pandas + olap куб из pandas или dask), вызов Rust через PyO3.
    - Make - сборка Rust библиотеки, компиляция .ui в .py, запуск тестов, упаковка.
    - Эволюционныймодуль - на Python, периодически тестирует текущий резервуар, при деградации создаёт новый через Rust.

#4. Требования к производительности
    - Резервуар до 300 нейронов работает быстрее 100 мс на шаг обучения на i5-3337U.
    - OLAP-куб до 500 тыс. записей - память <2GB.
    - График обновляется без зависаний (отдельный QThread для Rust-вычислений).
    - Файл .ui интерфейса
    - Скрипты для тестовых данных (финансы / логистика)
    - README.md с инструкцией по сборке и запуску
